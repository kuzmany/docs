---
description: How to use the Multiple Transport Plugin in Mautic.
---

<Callout type="note" emoji="🛠️">
  <b>Before you start:</b>  
  Add at least one transport in the "Multiple Transport" section. If something doesn't work, try clearing the Mautic cache.
</Callout>

<Steps>
  <Step title="Assign a Transport to an Email">
    <ol>
      <li>Open the email edit screen in Mautic.</li>
      <li>Find the "Transport" dropdown.</li>
      <li>Pick the transport you want for this email.</li>
      <li>Save the email.</li>
    </ol>
  </Step>
  <Step title="Use Contact Owner's Transport">
    <ol>
      <li>In the email edit screen, turn on "Use Owner Custom Transport".</li>
      <li>The email will use the transport set for the contact's owner (user).</li>
    </ol>
  </Step>
  <Step title="Assign a Transport to a User">
    <ol>
      <li>Go to the user profile edit screen.</li>
      <li>Find the "Transport" dropdown.</li>
      <li>Pick the transport for this user.</li>
      <li>Save the user.</li>
    </ol>
  </Step>
  <Step title="Add or Edit Transports">
    <ol>
      <li>Go to the "Multiple Transport" section in the Mautic admin menu.</li>
      <li>Click "New" to add a transport, or edit an existing one.</li>
      <li>Fill in the details (name, DSN, etc).</li>
      <li>Save.</li>
    </ol>
  </Step>
  <Step title="Remove or Reset a Transport">
    <ul>
      <li>To remove a transport from an email or user, set the dropdown to blank and save.</li>
      <li>If no transport is set, Mautic will use the default transport.</li>
    </ul>
  </Step>
  <Step title="Test a Transport">
    <ul>
      <li>When editing a transport, use the "Test" button to send a test email and check if it works.</li>
    </ul>
  </Step>
</Steps>

<Callout type="tip" emoji="💡">
  If a transport is not working, double-check the DSN and credentials.  
  If changes are not applied, clear the Mautic cache.  
  Make sure you have permission to manage transports.
</Callout>
